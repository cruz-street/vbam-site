# Vero Beach Adult Medicine — Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a new greenfield Next.js marketing site for Vero Beach Adult Medicine (VBAM), deployed on Cloudflare Pages, using the v0.4 VBAM brand palette and the exact workflow established by the brewer-international project.

**Architecture:** Static Next.js 15 App Router site (`output: 'export'`) with Tailwind CSS 4 custom design tokens, hosted on Cloudflare Pages. No CMS — content is authored directly in components. Design tokens are isolated in `globals.css` so the palette can be swapped to v0.5 when stakeholders lock the brand.

**Tech Stack:** Next.js 15 · Tailwind CSS 4 · TypeScript strict · pnpm · Web3Forms · Cloudflare Pages · Vitest (for any utility functions)

```markdown
New Next.js marketing site for VBAM, deployed on Cloudflare Pages.

## Rules

### 1. Brand Token Rule (Non-Negotiable)
All colors come from `globals.css` `@theme {}` Tailwind tokens (prefix `vbam-`).
Never use ad-hoc hex values in components. When v0.5 palette is approved, update

routes until the plan is reviewed.

### 3. SEO Gate (Non-Negotiable)
Five-point checklist must pass before DNS cutover:
- [ ] Every URL mapped in `app/public/_redirects`
- [ ] `metadata` export on every page (title, description, canonical)
- [ ] `app/sitemap.ts` generated and verified
- [ ] `app/robots.ts` correct
- [ ] JSON-LD on home + contact pages

### 4. Forms
Use Web3Forms for the contact/appointment form. Honeypot field required.
Env var: `NEXT_PUBLIC_WEB3FORMS_KEY`. No server required.

### 5. Context7 Rule
Before any library or infrastructure work, fetch current docs via Context7 MCP.
Do not rely on training data for Next.js, Cloudflare Pages, or Tailwind CSS 4.

### 6. Memory Rule
`.claude/memory/` files are committed. Update `MEMORY.md` after every session.

### 7. Changelog Rule
Update `[Unreleased]` in `CHANGELOG.md` after every feature. Keep a Changelog format.

### 8. Deployment Log Rule
Record every CF Pages deployment in `DEPLOYMENTS.md` with date, URL, commit, notes.

### 9. Decision Log Rule
Every architecture decision goes in `docs/decisions/site-decisions.md` (newest first).

### 10. Design Memory Rule (Non-Negotiable)
- `docs/design-system.html` — visual token + component reference
- `docs/screens.html` — canonical page wireframes
Before any UI work: open `docs/screens.html`. After any design change: update both files.

### 11. screens.html is Canonical (Non-Negotiable)
`docs/screens.html` is the single source of truth for layouts. Before adding any UI,
verify layout exists there. Never introduce a new layout without adding it first.

### 12. Design Commandments Gate (Non-Negotiable)
Before implementing any component, check all 7:
1. Clarity First — visitor understands the practice in 5 seconds
2. Performance — PageSpeed ≥ 90 mobile
3. SEO Baked In — semantic HTML, no div-soup headings
4. Image Intentionality — meaningful alt text on every image
5. Mobile Responsive — every component verified on mobile
6. YAGNI — no features beyond the 5 planned pages
7. Brand Consistency — only locked vbam- tokens

### 13. Refactor After It Works
No refactoring during feature work. Build it, make it work, then refactor.
```

```markdown
# VBAM Design System Spec

**Status:** v0.4 — in review. v0.5 will be the locked production version.

## Typography

### Display (headings, hero)
Font: Playfair Display
- Hero H1: 56px / 64px leading, font-weight 700, color: atlantic
- Section H2: 36px / 44px, font-weight 600, color: atlantic
- Card H3: 22px / 30px, font-weight 600, color: atlantic
- Italic H1 variant: font-style italic, used for brand name emphasis (e.g. "Adult Medicine")

### Body/UI (navigation, body text, labels)
Font: Inter
- Body: 16px / 26px, font-weight 400, color: atlantic (at 80% opacity) or #3D4F5C
- Small/label: 12px / 18px, font-weight 500, letter-spacing 0.08em, text-transform uppercase
- Nav: 14px / 20px, font-weight 500
- Button: 14px / 20px, font-weight 600

## Color Application

See locked tokens in CLAUDE.md. Application ratios from v0.4 brief:
- Atlantic 35% — backgrounds, text, nav
- Foam 25% — page background, card fills
- Sand 15% — section backgrounds, dividers
- Amber 10% — accents, hover states, icon fills
- Coral 8% — CTA buttons, energy accents
- Seagrass 4% — success states, secondary accents
- Tide 3% — light link states

## Component Patterns

### Buttons
- Primary: bg-vbam-atlantic text-white px-6 py-3 rounded-sm font-semibold hover:bg-vbam-tide
- Secondary (outline): border-2 border-vbam-atlantic text-vbam-atlantic hover:bg-vbam-foam
- CTA (coral): bg-vbam-coral text-white hover:bg-vbam-amber

### Section Structure
Each section: max-w-6xl mx-auto px-4 sm:px-6 lg:px-8, py-20 (desktop) py-12 (mobile)

### Trust signals
Small caps label in vbam-coral above section headings (e.g., "FROM THE FAMILIES WHO KNOW")

### Icon style
Thin line icons (stroke-width 1.5). Seagrass or Atlantic fill. No filled/solid icons on the main brand.

## Logo Usage

SVG inline. Two variants:
- Full lockup: mark + "Vero Beach Adult Medicine" + tagline "Sunrise to Shoreline Care"
- Mark only: sun/sea symbol
- Light version (on dark bg): use white fill
- Dark version (on light bg): atlantic + amber fills

## Spacing

8px base unit.
- xs: 4px, sm: 8px, md: 16px, lg: 24px, xl: 32px, 2xl: 48px, 3xl: 64px, 4xl: 96px

## Border Radius
Minimal: rounded-sm (2px) for buttons, rounded-md (6px) for cards. No pill shapes.
```
